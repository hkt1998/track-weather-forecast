本模块为 Next.js App Router 项目的顶层脚手架，不包含业务源码，仅承载运行时与开发期配置：
- `next.config.ts` 使用 TypeScript 导出空 `NextConfig`，当前未启用自定义路由/重写等扩展。
- `tsconfig.json` 开启 strict 模式、ES2017 target、bundler 解析，并通过 `@/*` 路径别名映射到项目根，配合 Next 插件生成 `.next/types`。
- `eslint.config.mjs` 采用 Flat Config，组合 `eslint-config-next/core-web-vitals` 与 `typescript` 规则，并用 `globalIgnores` 显式忽略 `.next/out/build` 等产物。
- `postcss.config.mjs` 接入 Tailwind CSS v4（通过 `@tailwindcss/postcss`），由 Next 在构建时调用。
- `package.json` 声明依赖：Next 16.2.10 + React 19.2.4 作为核心，Leaflet + react-leaflet 用于地图渲染，better-sqlite3 + @xmldom/xmldom + @tmcw/togeojson 表明后端侧具备本地 SQLite 数据库与 GML/XML 地理数据解析能力；devDependencies 锁定 TS 5、Tailwind v4、ESLint 9。
- 入口约定遵循 App Router：`app/page.tsx` 为默认页面，`.gitignore` 排除 `node_modules/.next/out` 等产物。
